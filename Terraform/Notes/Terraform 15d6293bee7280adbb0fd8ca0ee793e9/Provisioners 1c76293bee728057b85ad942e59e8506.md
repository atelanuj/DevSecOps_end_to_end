# Provisioners

# Points:

- Provisioners are used to execute commands, scripts on local and remote servers.
- Its a way of doing configuration management once the server from the terraform has been provisioned.
- **it should be used as a last resort,** as they don’t fit Terraform’s declarative model and can introduce fragility.
- you can add a `provisioner` block inside the `resource` block of a compute instance.
- if a provisioners fails the that resource becomes a tainted resource. means the terraform will destroy and recreate that resource.

---

# Why use the provisioners:

- **Bootstrap tasks**: install software, configure services on a new instance
- **File transfers**: copy config files or certificates to remote machines
- **Cleanup**: run commands before destroying a resource

---

# Types of provisioners

- File provisioners
    - copies file and directories from local to remote server
    
    ```hcl
    resource "aws_instance" "web" {
      # ... other configuration
    
      provisioner "file" {
        source      = "conf/myapp.conf"
        destination = "/etc/myapp.conf"
      }
    }
    ```
    
- Local-exec
    - Executes commands on the local machine where Terraform is running.
    
    ```hcl
    resource "aws_instance" "web" {
      # ... other configuration
    
      provisioner "local-exec" {
        command = "echo 'Instance created: ${self.public_ip}' >> instances.txt"
      }
    }
    ```
    
- Remote-exec
    - Executes commands on a remote machine over SSH (Linux) or WinRM (Windows). [connection block](https://developer.hashicorp.com/terraform/language/resources/provisioners/connection#example-usage)
    
    ```hcl
    resource "aws_instance" "web" {
      # ... other configuration
    
      provisioner "remote-exec" {
        inline = [
          "sudo apt-get update",
          "sudo apt-get install -y nginx",
          "sudo systemctl start nginx"
        ]
      }
    
    # to established the connection with remote server
      connection { 
        type        = "ssh"
        user        = "ubuntu"
        private_key = file("~/.ssh/id_rsa")
        host        = self.public_ip
      }
    }
    ```
    
    - Remote-exec requires a connection block to connect to the server
        
        ```hcl
        connection {
          type        = "ssh"
          user        = "admin"
          private_key = file("path/to/key")
          host        = self.public_ip
        }
        ```
        
- null_resource with Provisioners
    - When you need to run provisioners without being tied to a specific resource:
    
    ```hcl
    resource "null_resource" "example" {
      provisioner "local-exec" {
        command = "echo 'This runs independently'"
      }
    
      triggers = {
        always_run = timestamp()
      }
    }
    ```
    
- **Failure Handling**: Provisioners can be configured to continue on failure:
    
    ```hcl
    provisioner "remote-exec" {
      inline = ["command-that-might-fail"]
      
      on_failure = continue  # or "fail" (default)
    }
    ```
    
- **Destroy Provisioners**: Run tasks when resources are destroyed:
    
    ```hcl
    provisioner "local-exec" {
      when    = destroy
      command = "echo 'Resource being destroyed'"
    }
    ```
